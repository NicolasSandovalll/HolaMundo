Principales actividades

- Creación de ramas para cada integrante
- Todos los integrantes subieron su versión del programa HolaMundo en su respectiva rama
- Cada miembro realizó una modificación en su código agregando su nombre y apellido
- Se realizó un merge de todas las ramas

Conclusión

La actividad permitió que cada integrante trabajara de manera independiente en su propia rama, facilitando la gestión de cambios y evitando conflictos en el código. Durante el proceso de integración, se detectó un conflicto al intentar fusionar las ramas debido a diferencias en el código. Este conflicto fue resuelto manualmente, combinando los cambios de ambas ramas para preservar el trabajo realizado por todos los integrantes. Una vez resuelto el conflicto, se completó la fusión de todas las ramas en la rama principal.
